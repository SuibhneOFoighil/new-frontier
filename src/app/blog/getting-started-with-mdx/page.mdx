# Getting Started with MDX

In this post, we'll dive deeper into MDX and explore how to set it up in your Next.js project.

## Installation

First, install the necessary packages:

```bash
npm install @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
```

## Configuration

Update your `next.config.ts` file:

```typescript
import type { NextConfig } from "next";
import createMDX from '@next/mdx'

const nextConfig: NextConfig = {
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
};

const withMDX = createMDX({
  // Add markdown plugins here, as desired
})

export default withMDX(nextConfig);
```

## Custom Components

One of the most powerful features of MDX is the ability to use custom React components. Create an `mdx-components.tsx` file in your project root:

```typescript
import type { MDXComponents } from 'mdx/types'

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    h1: ({ children }) => (
      <h1 className="text-4xl font-bold mb-6">{children}</h1>
    ),
    // ... other components
    ...components,
  }
}
```

## Metadata Export

You can export metadata from your MDX files:

```javascript
export const metadata = {
  title: 'My Post Title',
  author: 'Author Name',
  date: '2024-01-20'
}
```

This metadata can then be imported and used in other components for generating blog indexes, SEO tags, and more.

## Best Practices

### 1. File Organization

Organize your MDX files in a logical structure:

```
src/
  app/
    blog/
      post-1/
        page.mdx
      post-2/
        page.mdx
```

### 2. Styling

Use Tailwind CSS classes in your MDX components for consistent styling across your blog.

### 3. Performance

MDX files are compiled at build time, making them very fast to serve. Take advantage of Next.js's static generation features.

## Advanced Features

### Syntax Highlighting

For better code highlighting, you can add plugins like `rehype-highlight`:

```bash
npm install rehype-highlight
```

### Math Support

Add math support with `rehype-katex`:

```bash
npm install rehype-katex remark-math
```

## Conclusion

MDX provides a powerful way to create rich, interactive blog content while maintaining the simplicity of Markdown. With Next.js, you get excellent performance and developer experience out of the box.
